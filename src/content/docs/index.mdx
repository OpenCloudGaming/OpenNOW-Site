## Quick start

<Steps>
1. **Download** the latest release from [GitHub Releases](https://github.com/OpenCloudGaming/OpenNOW/releases).

2. **Install** the package for your platform:
   - **Windows x64** — NSIS installer or portable `.exe` with native GStreamer runtime bundled
   - **Windows ARM64** — NSIS installer or portable `.exe`
   - **macOS** — `.dmg` or `.zip` for Intel and Apple Silicon
   - **Linux** — `.AppImage` or `.deb` for x64 and ARM64
   - **iOS** — TestFlight/prototype availability is linked from the project README when active

3. **Sign in** with your NVIDIA GeForce NOW account.

4. **Play** — browse games, pick a queue/server when needed, launch a session, and adjust stream settings.
</Steps>

<LinkCard title="Full setup guide" description="Download, install, first launch, or build from source" href="/guides/getting-started/" />

<Aside type="caution" title="Native streamer is experimental">
OpenNOW defaults to Chromium/WebRTC. The Rust/GStreamer native streamer is opt-in, may hit platform-specific bugs or fallbacks, and should be enabled only if you are prepared to report native-streamer issues on [GitHub Issues](https://github.com/OpenCloudGaming/OpenNOW/issues) or [Discord](https://discord.gg/8EJYaJcNfD).
</Aside>

## Features

| Category | What's included |
| --- | --- |
| **Streaming** | Chromium WebRTC renderer path plus optional experimental Rust/GStreamer native streamer child process |
| **Native video** | Experimental GStreamer `webrtcbin` backend with Windows D3D12/D3D11, macOS VideoToolbox, Linux VAAPI/V4L2, and software fallback paths |
| **Stream quality** | H.264/H.265/AV1 preferences, color-quality normalization, resolution up to 4K, FPS up to 360, bitrate cap, region, L4S, Cloud G-Sync/VRR setting |
| **Authentication** | Browser OAuth + PKCE with persistent local auth state and token refresh |
| **Game library** | Personal library, public catalog, search, queue server selector, and displayed wait-time/session state |
| **Input** | Pointer lock, keyboard layout setting, mouse sensitivity/acceleration, gamepad support, clipboard paste, native input status where available |
| **Session UI** | Stats-on-launch option, anti-AFK indicator, session timer, fullscreen shortcuts, update checks, Discord Rich Presence |
| **Capture** | Renderer screenshots and MediaRecorder video capture saved locally with thumbnails and in-app gallery |
| **Controller mode** | Controller-first library, UI sounds, background animations, theme style/color, and game backdrops |
| **Cross-platform** | Windows x64/ARM64, macOS x64/arm64, Linux x64/arm64, plus the separate iOS prototype/TestFlight link when available |

## Architecture at a glance

OpenNOW is an Electron app with an optional native child-process layer:

| Layer | Responsibility |
| --- | --- |
| **Main** (Node.js) | OAuth, CloudMatch sessions, queue/session state, WebSocket signaling, settings, updater, Discord RPC, media storage, native streamer lifecycle |
| **Preload** | Typed `window.openNow` IPC bridge via `contextBridge` |
| **Renderer** (React) | UI, library/controller mode, WebRTC renderer client, input capture, stats overlay, capture controls |
| **Native streamer** (Rust, optional, experimental) | JSON-lines protocol v2 child process; GStreamer offer/answer, ICE, data channels, decode/render, input readiness, and native diagnostics |

The active source lives in `opennow-stable/src/`, shared contracts in `opennow-stable/src/shared/`, and the Rust streamer in `native/opennow-streamer/`.
