OpenNOW is an open-source desktop client for GeForce NOW. It keeps the GFN session lifecycle in Electron, streams through Chromium WebRTC by default, and includes an experimental Rust/GStreamer native streamer path for Windows testing.

## Quick start

<Steps>
  <Step>
    **Download** the latest release from [GitHub Releases](https://github.com/OpenCloudGaming/OpenNOW/releases).
  </Step>
  <Step>
    **Install** the package for your platform: Windows `.exe`, macOS `.dmg`/`.zip`, Linux `.AppImage`/`.deb`, or the iOS [TestFlight beta](https://testflight.apple.com/join/u1XPJKH2).
  </Step>
  <Step>
    **Sign in** with your NVIDIA GeForce NOW account through the browser OAuth flow.
  </Step>
  <Step>
    **Play** by browsing games, picking a queue/server when needed, launching a session, and adjusting stream settings.
  </Step>
</Steps>

<Callout type="warning" title="Native streamer is experimental">
OpenNOW defaults to Chromium/WebRTC. The Rust/GStreamer native streamer is opt-in on Windows, currently gated off on macOS/Linux, and can fall back to the web streamer when platform media support is missing.
</Callout>

## Features

| Category | What's included |
| --- | --- |
| **Streaming** | Chromium WebRTC renderer path plus optional experimental Rust/GStreamer native streamer child process on Windows |
| **Native video** | Experimental GStreamer `webrtcbin` backend with Windows D3D12/D3D11; macOS/Linux backend paths exist in code but native mode is currently gated off in the app |
| **Stream quality** | H.264/H.265/AV1 preferences, color-quality normalization, resolution up to 4K, FPS up to 360, bitrate cap, region, L4S, Cloud G-Sync/VRR setting |
| **Authentication** | Browser OAuth + PKCE with persistent local auth state and token refresh |
| **Game library** | Personal library, public catalog, search, queue server selector, displayed wait-time/session state, and membership-tier launch-error copy |
| **Input** | Pointer lock, keyboard layout setting, mouse sensitivity/acceleration, gamepad support, clipboard paste, native input status where available |
| **Session UI** | Stats-on-launch option, anti-AFK indicator, session timer, desktop accent color, app localization, fullscreen shortcuts, update checks, Discord Rich Presence |
| **Capture** | Renderer screenshots and MediaRecorder video capture saved locally with thumbnails and in-app gallery |
| **Controller mode** | Controller-first library, UI sounds, background animations, theme style/color, and game backdrops |
| **Cross-platform** | Windows x64/ARM64, macOS x64/arm64, and Linux x64/arm64 desktop packages, plus an iOS TestFlight beta |

## Architecture at a glance

| Layer | Responsibility |
| --- | --- |
| **Main** (Node.js) | OAuth, CloudMatch sessions, queue/session state, WebSocket signaling, settings, updater, Discord RPC, media storage, native streamer lifecycle |
| **Preload** | Typed `window.openNow` IPC bridge via `contextBridge` |
| **Renderer** (React) | UI, library/controller mode, WebRTC renderer client, input capture, stats overlay, capture controls |
| **Native streamer** (Rust, optional, experimental) | JSON-lines protocol v2 child process; GStreamer offer/answer, ICE, data channels, decode/render, input readiness, and native diagnostics |

The active source lives in `opennow-stable/src/`, shared contracts in `opennow-stable/src/shared/`, and the Rust streamer in `native/opennow-streamer/`.
